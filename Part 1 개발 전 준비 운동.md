### 02. ios 개발 학습 방향

- 계속 개발을 함으로써 습관을 들이고 점진적으로 발전시켜 나가야함
- 4가지 법칙
    - 분명하게 하기 (시간, 장소)
    - 매력적이어야 한다.
        - 살면서 불편했던 점 또는 개발하고 싶은 어플 생각을 기록해두기
        - 단순히 돈을 벌고 싶다는 마인드 보다 내가 하고 싶은 것을 하려고 노력하기
    - 쉬워야 한다.
        - 지치지 않기 위해 쉬운 것부터 시작하여 점차적으로 늘려가는 것이 중요하다.
    - 만족스러워야 한다.
- 학습 퀄리티를 높이는 3가지 방법
    - 그룹 이루기
    - 성취하기 쉬운 목표 설정
    - 시간 정하기

### 03. 네카라쿠배당토 모집요강 뿌시기

- 모집요강 뿌실 때 봐야 하는 3가지 요소
    - 제품
    - 기술
    - 문화
- 공통 요소
    - 제품
        - 글로벌
    - 기술
        - swift (basic + advanced)
        - swiftUI, UIKit (storyboard/ auto-layout/ 코드로 UI)
        - 네트워크
            - Restful API
        - OOP (objective oriented programming)
            - 모듈화
            - 디자인 패턴, MVVM
            - 클린 아키텍쳐
        - Reactive Programming
            - Combine
            - Rx
            - 함수형 프로그래밍
        - 생산성 (자동화)
            - CI/CD
            - 테스트 자동화
        - 앱 접근성
    - 문화
        - 개발문화
            - 코드리뷰
            - 리팩토링
            - TDD
            - 페어 코딩
            - 스프린트/ 데일리 스크럼/ 회고
        - 커뮤니케이션
        - 주도적
        - 성장
